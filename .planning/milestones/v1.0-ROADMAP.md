# Milestone v1.0 Archive

## Project
RCL / MNiSW Newsletter Quality Upgrade

## Version
v1.0

## Status
Shipped and archived on 2026-03-14.

## Milestone Summary
This milestone transformed the local RCL/MNiSW newsletter workflow from a technically working prototype into a production-ready pipeline with AI-based project summaries, polished newsletter presentation, and end-to-end quality validation on real email sends.

## Scope

### Phase 1
AI summary pipeline for projects

Goal:
Zastąpić obecne heurystyczne streszczenia warstwą AI opartą o treść uzasadnienia i metadane projektu.

Scope:
- zdefiniować dane wejściowe do AI,
- dodać node lub pod-flow AI,
- uzgodnić format odpowiedzi AI,
- zintegrować wynik z obecnym scoringiem i buildem maila,
- dodać sensowny fallback bez śmieciowego tekstu.

Outcome:
Completed.

### Phase 2
Newsletter UI and copy polish

Goal:
Poprawić layout i copy newslettera w HTML/text.

Scope:
- odstęp między kaflami statystyk w nagłówku,
- zmiana copy sekcji na `Najnowsze projekty w RCL`,
- poprawa spacingu w metadanych projektu,
- poprawa odstępów i rytmu typograficznego kart,
- stopka z brandingiem i danymi autora.

Outcome:
Completed.

### Phase 3
End-to-end validation and content review

Goal:
Wprowadzić twardą walidację końcowej jakości newslettera.

Scope:
- test pełnego przebiegu aż do wysyłki,
- weryfikacja treści wygenerowanych opisów,
- korekta promptu AI i copy tam, gdzie mail nadal brzmi zbyt formalnie lub zbyt mało informacyjnie,
- sprawdzenie kilku realnych projektów z różną jakością dokumentów.

Outcome:
Completed.

## Key Accomplishments
- Added AI-generated `summary`, `author`, `keyChanges`, and `sourceQuality` to the workflow using the existing Groq integration.
- Improved source extraction so project documents stored mainly in Word tables are now usable as AI input.
- Added quality guards that reject title-like or overly generic AI summaries and fall back to more concrete outputs.
- Polished the newsletter HTML/text with better header spacing, clearer metadata rows, updated section naming, and a branded footer.
- Removed a false-positive MNiSW classification caused by long stakeholder lists in raw legislative documents.
- Verified the system with real Resend deliveries across all three phases.
